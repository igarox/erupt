## Product Context

- **What this is:** An Obsidian plugin that extracts structured notes from AI chat transcripts into a living vault wiki.
- **Who it's for:** Obsidian power users βÄî founders, researchers, writers, academics βÄî who generate significant intellectual output in AI chats and lose it.
- **Platform:** Obsidian desktop (Windows, macOS, Linux). `isDesktopOnly: true`.
- **Project type:** Plugin UI βÄî settings panel, modals, status bar item. All surfaces are constrained by Obsidian's plugin API and theming system.
- **Brand relationship:** Erupt is a Slipstream product. External-facing design treats Erupt as standalone. The Slipstream connection surfaces only in the "by Slipstream" byline treatment.
- **Family:** Erupt is part of the Slipstream geological product family βÄî Magma (underground foundation/shadow vault), Lava (agentic generation plugin), Erupt (extraction plugin). All live inside Obsidian, which is itself volcanic glass. The family shares a geological origin story, not a borrowed brand.

---

## Aesthetic Direction

- **Direction:** Obsidian-native minimalism + geological heat
- **Decoration level:** Minimal on structure, intentional on magic surfaces only
- **Mood:** Quiet and trustworthy at rest. The settings panel disappears into Obsidian. The modal surfaces heat βÄî when Erupt is working or when it names itself, something glows. Users feel like they opened a hatch above something pressurized.
- **Core constraint:** Erupt does not own its canvas. The plugin lives inside Obsidian's theming system. Design expression is limited to: the icon, the accent color, the modal, the status bar copy, and the shimmer treatments. These four surfaces do all the work. Everything else defers to the host.

## Geological Family Color Map

Each product in the family has a single accent that represents its geological position βÄî temperature and depth increasing as you go deeper:

| Product | Accent | Meaning |
|---------|--------|---------|
| Erupt | `#E85D26` | Ember orange βÄî glowing rock at the surface, cooling in air |
| Lava | `#C8390A` | Molten red-orange βÄî active, dangerous, just broken through |
| Magma | `#8B1A1A` | Dark crimson βÄî underground, pressurized, not yet visible |

These form a temperature gradient from surface-bright to underground-dark. Consistent logic, distinct identities. Lava and Magma tokens are listed here for family coherence βÄî their authoritative tokens live in their own DESIGN.md files.

## Color βÄî Erupt

- **Approach:** Restrained βÄî one accent over Obsidian's native token layer
- **Accent:** `#E85D26` βÄî ember orange
- **Accent hover:** `#CC4D1E` βÄî 15% darker
- **Accent tint:** `#FEF0EA` βÄî used for subtle backgrounds in light mode
- **Success:** `#16A34A` βÄî completion state
- **Warning:** `#D97706`
- **Error:** `#DC2626`

### CSS Variables

Map Erupt accent onto Obsidian's system. Do NOT hardcode colors directly in component CSS.

```css
.theme-dark, .theme-light {
  --erupt-accent:       #E85D26;
  --erupt-accent-hover: #CC4D1E;
  --erupt-accent-light: #FEF0EA;
  --erupt-success:      #16A34A;
  --erupt-warning:      #D97706;
  --erupt-error:        #DC2626;
}
```

Use `--interactive-accent` from Obsidian for interactive elements so users who override the accent in their theme get expected behavior. Reserve `--erupt-accent` for Erupt-specific branding surfaces (the icon tint, the modal heading, shimmer base color).

**Dark mode:** Obsidian handles background/surface inversion. Erupt tokens are accent-only and remain stable across themes. The shimmer effects are designed for dark mode first βÄî on light themes they remain legible but less dramatic.

## Typography

- **All text:** System font stack via Obsidian's `var(--font-interface)` βÄî do NOT import external fonts. Importing Google Fonts from a plugin conflicts with user themes.
- **Data surfaces** (transcript turn counts, version history timestamps, extraction progress fractions): `var(--font-monospace)` βÄî Obsidian exposes this as a CSS variable; use it directly.
- **Code/prompt previews:** `var(--font-monospace)` at 0.85em, `var(--background-modifier-form-field)` background.

### Type Scale

Use Obsidian's own type scale via CSS variables rather than hardcoded px values:

| Role | Variable | Usage |
|------|----------|-------|
| Modal heading | `var(--h2-size)` | "Extract Notes" heading in modal |

| Supporting | `var(--font-ui-small)` | Descriptions, help text, status bar |

## Shimmer Treatments

Two distinct shimmer treatments. They have different visual languages and different meanings. Never use them interchangeably.

### `shimmer-brand` βÄî "by Slipstream" byline

Full visible spectrum, mouse-position-reactive. Feels like a holographic sticker or diffraction grating shifting as you look at it from different angles. Requires a small JS helper βÄî see implementation below.

**When to use:** Only on the literal text "by Slipstream" wherever it appears as a subtitle (modal footer, settings panel header, about section).

**Visual character:** Full spectrum violetβÜ£red, gradient angle and position shift with mouse. Slow, physical, no looping animation. The text catches light differently depending on where the cursor is in the modal.

```css
.slipstream-byline {
  --mx: 0.5;
  --my: 0.5;
  background: linear-gradient(
    calc(60deg + (var(--mx) - 0.5) * 120deg),
    #FF0055,
    #FF6200,
    #FFE500,
    #00FF94,
    #00C8FF,
    #0066FF,
    #6B00FF
  );
  background-size: 200% 200%;
  background-position: calc(var(--mx) * 100%) calc(var(--my) * 100%);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  background-clip: text;
  transition: background-position 0.05s ease-out;
  font-size: var(--font-ui-small);
  font-weight: 500;
  letter-spacing: 0.04em;
  text-transform: uppercase;
}

@media (prefers-reduced-motion: reduce) {
  .slipstream-byline {
    background: linear-gradient(
      90deg,
      #FF0055, #FF6200, #FFE500, #00FF94, #00C8FF, #0066FF, #6B00FF
    );
    background-size: 100% 100%;
    -webkit-background-clip: text;
    background-clip: text;
    transition: none;
  }
}
```

**JS helper βÄî add to any modal that shows the byline:**

```typescript
private attachShimmerListener(modalEl: HTMLElement, bylineEl: HTMLElement): void {
  modalEl.addEventListener('mousemove', (e: MouseEvent) => {
    const rect = modalEl.getBoundingClientRect();
    bylineEl.style.setProperty('--mx', ((e.clientX - rect.left) / rect.width).toFixed(3));
    bylineEl.style.setProperty('--my', ((e.clientY - rect.top) / rect.height).toFixed(3));
  });
}
```

Call from the modal's `onOpen()` after rendering the byline element. No cleanup needed βÄî the listener is scoped to the modal element which is garbage collected when the modal closes.

### `shimmer-product` βÄî magic feature names

Geological loop animation. Cycles through Erupt's accent palette: ember orange βÜ£ gold βÜ£ pale yellow βÜ£ cool mint βÜ£ back to ember. Reads as thermal, not prismatic. No JS required.

**When to use:** On key feature names in modal headings βÄî "Extract Notes," "Semantic Search," "Version History" βÄî and on the word "Erupt" when used as a standalone heading (not body text). Do not apply to every heading. Reserve this for the one or two moments per modal where Erupt names its signature capability.

## Motion

- **Approach:** Minimal-functional + two shimmer treatments (above)
- **Easing:** `ease-out` for enter transitions, `ease-in` for exit, `ease-in-out` for state changes
- **Duration:** Micro 80ms / Short 150ms / Medium 250ms

Status bar state changes: 150ms cross-fade on text. No bounce, no slide βÄî just opacity. Obsidian's own UI is flat and immediate; jarring motion in the status bar will feel wrong.

## Plugin Surfaces

### Settings Panel

Two distinct states based on plan tier. Use Obsidian's `Setting` renderer for all fields βÄî do not build custom input components.

**Free (BYOK) state:**
- Section header: "API Key" βÄî `Setting` with name + description
- Input: API key field, `type="password"`, placeholder "sk-ant-..."
- Below input: small help text in `--text-muted` βÄî "Your key is stored locally and never sent to Slipstream."
- Upgrade nudge: one line, `--text-muted`, a single unobtrusive link βÄî "Upgrade to Cloud for automatic key management."

**Cloud (paid) state:**
- Section header: "Account" βÄî shows authenticated email
- Button: "Sign out" βÄî `mod-warning` class
- No API key field visible

**Shared settings (both states):**
- Toggle: "Show stubs and TODOs by default"
- Toggle: "PII sanitization" (always on, shown as on + locked with explanation)
- Button: "View version history" βÄî secondary style
- Byline at bottom of settings: "by Slipstream" with `shimmer-brand` treatment

### Status Bar

Minimal text, monospace feel. States:

| State | Copy | Style |
|-------|------|-------|
| Idle | `Erupt` | `--text-muted`, standard weight |
| Processing | `Erupt: turn X/YβÄΠ` | `--text-normal`, X and Y in `var(--font-monospace)` |
| Complete | `Erupt: done β€™` | `--text-success` for 2s, then fades back to idle |
| Error | `Erupt: error` | `--text-error`, stays until dismissed or next run |

`aria-live="polite"` on the status bar element. The status bar item is a live region βÄî screen readers announce state changes without focus.

### Completion Modal

The primary emotional moment βÄî the one surface where design expression is highest.

- Modal title: "by Slipstream" βÄî `shimmer-brand`, small, uppercase, letter-spaced. Not the product name. The product name is in the content.
- Heading: "Erupt" βÄî `shimmer-product`, `var(--h2-size)`
- Sub-heading: note count, folder path in `--text-muted`
- Body: summary of what was extracted (N notes, M stubs, K TODOs)
- If clarifying questions exist: a scrollable list, each with an inline input
- Primary CTA: "Done" βÄî `mod-cta`
- Secondary: "Open folder" βÄî standard button

### Session Picker Modal (unlinked note trigger)

Minimal. No shimmer βÄî this is a utility modal, not a magic moment.

- Title: "Which session?" βÄî plain, `var(--h3-size)`
- List of sessions: keyboard-navigable (`βÜë` / `βÜ™` / `Enter` / `Escape`)
- Each row: session title + date in `--text-muted` + turn count in monospace
- Focus ring: `--interactive-accent` outline
- `role="listbox"` on the container, `role="option"` on each row, `aria-selected` on focused item

### Upgrade Modal

Triggered only on deliberate action (clicking upgrade link in settings) βÄî never auto-shown mid-flow.

- No shimmer on this modal (avoid feeling pushy)
- Title: "Cloud Plan" βÄî plain
- One-line value prop, two feature bullets, price
- Primary CTA: "Upgrade" βÄî `mod-cta`
- Secondary: "Not now" βÄî plain text link, not a button

## Icon

**Concept:** A stylized eruption cross-section βÄî a teardrop or inverted flame shape, cracked at the apex with a light source visible inside. Geometric, flat-filled. Same visual grammar as Obsidian's own gem icon (multifaceted, angular, minimal) but representing the geological event rather than its end product.

**Palette:** Two fills βÄî a dark base (`#1A1A1A` or transparent) and the accent `#E85D26` for the interior light source. One thin highlight stroke in `#F5A623` on the crack edge.

**Format:** SVG, 16Ο½16 for ribbon and status bar, 32Ο½32 for settings header. No raster. Must read at 16px with no label.

## Accessibility

- `aria-live="polite"` on status bar element
- All modals: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading
- Session picker: `role="listbox"` / `role="option"` / `aria-selected`
- Focus trap in all modals βÄî `Tab` cycles within, `Escape` closes
- Keyboard navigation for all interactive elements βÄî no mouse-only interactions
- Contrast: all text against Obsidian's theme backgrounds must meet WCAG AA (4.5:1 body, 3:1 large text)
- `shimmer-brand` and `shimmer-product`: both include `prefers-reduced-motion` fallbacks (static gradient, no animation)
